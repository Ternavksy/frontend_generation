1 Используемые термины
Термин Определение
Маска (Mask) Растровое изображение, где пиксели
соответствуют принадлежности к
определенному классу объекта
Сегментация Процесс разделения изображения на
множественные сегменты для
упрощения представления
Детекция Процесс обнаружения объектов на
изображении и выделения их
ограничивающими рамками (Bounding
Box)
Bounding Box (Box) Прямоугольная рамка, описывающая
координаты обнаруженного объекта
Модель ИИ Обученная нейронная сеть (например,
SAM2, YOLO-World), используемая для
автоматической разметки
Рабочая область (Workspace) Основной интерфейс пользователя для
просмотра изображения и
редактирования разметки
Класс объекта Категория, к которой принадлежит
размеченный объект (например,
«автомобиль», «пешеход»)
SAHI Slicing Aided Hyper Inference, техника
для улучшения детекции мелких
объектов
WebSocket (WS) Протокол связи для организации
очереди задач между сервером и
моделями

Система включает следующие программные компоненты:
1. Фронтенд-приложение (React / JavaScript):
1.a. Модуль авторизации и управления подпиской
1.b. Рабочая область разметки (на базе архитектуры CVAT с доработками)
1.c. Панель управления моделями и датасетами
1.d. Личный кабинет с историей задач и лимитами
2. Бэкенд-сервисы (Python / FastAPI):
2.a. Основной сервер (API, аутентификация, управление очередью WS)
2.b. Сервис обработки изображений (валидация, конвертация форматов)
2.c. Сервис управления задачами сегментации/детекции
2.d. Шлюз для взаимодействия с локальными сервисами моделей
3. Сервисы моделей (Python / Docker):
3.a. Сервис сегментации (SAM2)
3.b. Сервис детекции (YOLO-World, GroundingDino + SAHI)
3.c. Изолированное развертывание на локальных портах
4. База данных (PostgreSQL):
4.a. Схема хранения пользователей, проектов, изображений, масок и
метаданных
4.b. Механизм учета лимитов (тарификация)
5. Инфраструктурные компоненты:
5.a. Файловое хранилище (локальное или S3-compatible)
5.b. Система логирования и мониторинга очередей
3 Назначение и цели создания системы
3.1 Цели создания системы
Система создается для ускорения и повышения качества процесса разметки данных
для компьютерного зрения путем:
1. Автоматической генерации масок и bounding box с использованием
современных моделей ИИ (SAM2, YOLO-World)
2. Предоставления продвинутых инструментов ручного редактирования масок
(слои, прозрачность, стирание)
3. Сравнения результатов работы различных моделей и выбора наилучшего
4. Организации тарификации ресурсов (ограничение на форматы, количество
кадров, параллелизм)
3.2 Задачи, решаемые при помощи системы
Для Анотатора:
1. Загрузка наборов данных (изображения + маски)
2. Автоматический прогон данных через доступные модели
3. Выбор лучших результатов и классов для сохранения
4. Редактирование масок: работа с классами по отдельности, инверсия фона,
ручная дорисовка
5. Разделение склеенных сегментов
6. Управление bounding box (удаление пересекающихся объектов) Для
Администратора:
7. Управление пользователями и тарифными планами
8. Добавление и настройка локальных моделей
9. Мониторинг нагрузки на сервисы моделей
10. Контроль лимитов использования (TIFF, количество кадров)
4 Требования к системе и программному обеспечению
4.1 Требования к программному обеспечению системы
Система реализуется по архитектуре клиент-сервер с микросервисной обработкой
моделей:
1. Серверная часть: Python 3.10+, FastAPI, SQLAlchemy, Celery/RQ (для
очередей)
2. Клиентская часть: React 18+, TypeScript, Konva.js или Fabric.js (для canvas),
Tailwind CSS
3. База данных: PostgreSQL 14+
4. Сервисы моделей: Docker-контейнеры, доступные только внутри локальной
сети сервера
5. Внешние зависимости:
5.a. Библиотеки для работы с изображениями (Pillow, OpenCV, Tiktoken)
5.b. Модели: SAM2, YOLO-World, GroundingDino
5.c. Протокол WebSocket для передачи задач в очередь моделей
4.2 Внутренние требования
1. Хранение конфигурации моделей и ключей доступа в переменных
окружения (.env)
2. Изоляция сервисов моделей: порты моделей не должны быть доступны
извне, только для основного сервера
3. Проверка прав доступа к функциям тарификации (TIFF, параллелизм) на
стороне сервера
4. Асинхронная обработка запросов через WebSocket очередь
4.3 Общие требования к оформлению и верстке интерфейса
1. Интерфейс рабочей области разметки базируется на принципах CVAT
(панель инструментов слева/сверху, холст по центру, слои справа)
2. Единый корпоративный стиль
3. Адаптивная верстка (поддержка разрешений от 1920x1080)
4. Темная и светлая тема для рабочей области (для удобства работы с масками)
5. Наличие индикаторов загрузки при обработке моделями
4.4 Требования к численности и квалификации персонала
1. Администратор системы: 1 человек, навыки управления Docker-
контейнерами и моделями
2. Анотатор: Не требует специальной технической подготовки, работа через
веб-интерфейс
3. Технический специалист: 1 человек для поддержки инфраструктуры (GPU
серверы)

5 Структура системы
Система реализует многоуровневую архитектуру:
1. Уровень представления (Presentation Layer): React-приложение (Рабочая
область разметки, Личный кабинет)
2. Уровень приложения (Application Layer): 2.a. API Gateway
(маршрутизация запросов, аутентификация) 2.b. Менеджер очередей
(WebSocket) 2.c. Сервис бизнес-логики (тарификация, валидация лимитов)
3. Уровень моделей (Model Layer): 3.a. Локальные сервисы (SAM2, YOLO,
GroundingDino) на отдельных портах 3.b. Интеграция SAHI для детекции
4. Уровень данных (Data Layer): PostgreSQL, Файловое хранилище

10 Описание компонентов системы
10.1 Страница авторизации
1. Поля ввода: Email, Пароль
2. Кнопки: «Войти», «Восстановить пароль»
3. Ссылка на регистрацию (при открытой регистрации)
4. Авторизация через сторонние сервисы
10.2 Личный кабинет и Тарифы
1. Отображение текущих лимитов (кол-во кадров, форматы, параллелизм)
2. История оплат и расширений
3. Профиль пользователя (ФИО, организация)
10.3 Загрузка датасета
1. Drag-and-drop зона для загрузки изображений (JPG, PNG)
2. Индикатор поддержки TIFF (доступен только при активной подписке)
3. Валидация размера файлов и количества кадров согласно тарифу
4. Прогресс-бар загрузки и предварительной обработки
10.4 Рабочая область разметки (Annotation Workspace)
1. Холст (Canvas):
1.a. Отображение исходного изображения
1.b. Наложение масок сегментации с регулируемой прозрачностью (ползунок
0–100%)
1.c. Режим «Черный/Белый фон»: переключение фона маски для контраста
(белый — класс, черный — фон)
2. Панель инструментов:
2.a. Инструмент «Ластик»: стирание белого (маски) с проявлением основного
изображения
2.b. Инструмент «Кисть»: закрашивание черным/белым для коррекции маски
конкретного класса (возможно редактирование через множество точек для
данного сегмента)
2.c. Инструмент «Разделение сегментов»: разрезание склеенных объектов
одного класса
2.d. Инструмент «Детекция»: рисование/редактирование Bounding Box
3. Панель слоев и классов:
3.a. Список классов с чекбоксами видимости
3.b. Возможность работы с каждым классом по отдельности (изоляция слоя)
3.c. Сравнение масок: режим «Split View» или переключение между
результатами разных моделей
3.d. Установка приоритета маски вручную при наложении классов
4. Взаимодействие с детекцией:
4.a. При наведении мыши на вложенный объект (внутри другого Box)
4.b. Комбинация клавиш Ctrl + Click удаляет выбранный Bounding Box
внутреннего объекта
5. Панель моделей:
5.a. Выбор модели для прогона (SAM2, YOLO-World и др.)
5.b. Кнопка «Запустить авто-разметку»
5.c. Индикатор прогресса обработки очереди через WebSocket соединение
10.5 Управление моделями (Администратор)
1. Список развернутых локальных моделей
2. Настройка портов и параметров доступа
3. Мониторинг статуса (Health Check) сервисов моделей
4. Панель поддерживается через сервер с помощью FastAPI Admin
10.6 Сравнение и выбор результатов
1. Интерфейс для просмотра результатов разных моделей на одном
изображении
2. Механизм выбора «Лучшего результата» для сохранения в датасет
3. Фильтрация по классам (оставить только выбранные классы)
11 Функциональность системы
11.1 Для Анотатора
1. Загрузка изображений (JPG, PNG бесплатно; TIFF при оплате)
2. Автоматический запуск сегментации/детекции через очередь WS
3. Редактирование масок:
3.a. Переключение прозрачности маски
3.b. Стирание частей маски (режим ластика)
3.c. Дорисовка маски (режим кисти)
3.d. Разделение склеенных объектов
4. Редактирование детекции:
4.a. Удаление пересекающихся Box через Ctrl + Click
4.b. Использование SAHI для улучшения детекции мелких объектов
5. Выбор лучших классов и сохранение результата
11.2 Для Администратора
1. Развертывание моделей локально (SAM2, YOLO-World, GroundingDino)
2. Настройка ограничений тарифов (лимит кадров, параллельная обработка)
3. Управление очередью задач (приоритизация, отмена)
4. Мониторинг нагрузки на GPU
11.3 Бизнес-логика и Тарификация
1. Ограничение форматов: Проверка расширения файла при загрузке (TIFF
блокируется для бесплатных аккаунтов)
2. Ограничение ресурсов: Лимит на количество одновременных задач
обработки моделей
3. Очередь задач: WS соединение управляет отправкой изображений на
свободные порты моделей
4. Сохранение версий: Возможность отката к предыдущей версии разметки
12 Структура базы данных
12.1 Основные таблицы и поля
Таблица users (пользователи):
1. id UUID PK
2. email VARCHAR(255) UNIQUE
3. password_hash VARCHAR(255)
4. role ENUM('annotator', 'admin', 'manager')
5. subscription_plan VARCHAR(50) (free, pro, enterprise)
6. limits_json JSONB (лимиты на TIFF, кадры, параллелизм)
Таблица projects (проекты):
1. id UUID PK
2. name VARCHAR(255)
3. created_by_id UUID FK → users.id
4. created_at TIMESTAMP
Таблица datasets (наборы данных):
1. id UUID PK
2. project_id UUID FK → projects.id
3. name VARCHAR(255)
4. status ENUM('uploading', 'ready', 'processing')
Таблица images (изображения):
1. id UUID PK
2. dataset_id UUID FK → datasets.id
3. file_path VARCHAR(500)
4. width INTEGER
5. height INTEGER
6. format VARCHAR(10) (jpg, png, tiff)
Таблица models (модели):
1. id UUID PK
2. name VARCHAR(100) (SAM2, YOLO-World)
3. type ENUM('segmentation', 'detection')
4. endpoint_url VARCHAR(255) (локальный порт)
5. is_active BOOLEAN
Таблица annotations (разметка):
1. id UUID PK
2. image_id UUID FK → images.id
3. model_id UUID FK → models.id (NULL если вручную)
4. class_name VARCHAR(100)
5. annotation_type ENUM('mask', 'bbox')
6. data JSONB (координаты полигона или маски)
7. priority INTEGER (приоритет маски)
8. is_selected BOOLEAN (выбрано как лучший результат)
Таблица tasks (задачи обработки):
1. id UUID PK
2. image_id UUID FK → images.id
3. model_id UUID FK → models.id
4. status ENUM('queued', 'processing', 'completed', 'failed')
5. created_at TIMESTAMP
